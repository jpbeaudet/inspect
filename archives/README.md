# archives/

This folder is **historical**. It holds the implementation plans,
design "bibles", audit checklists, and field-pitfall catalogs that
drove the v0.1.0 build. They are kept for context — they are not the
maintained surface of the project.

For current documentation, use:

- [`README.md`](../README.md) — overview and install.
- [`docs/MANUAL.md`](../docs/MANUAL.md) — the user manual.
- [`docs/RUNBOOK.md`](../docs/RUNBOOK.md) — the maintainer / on-call runbook.
- [`docs/RELEASING.md`](../docs/RELEASING.md) — release flow.
- [`CHANGELOG.md`](../CHANGELOG.md) — per-release changes.
- `inspect help <topic>` — the in-binary, offline manual.

If something in `archives/` contradicts a doc above, the doc above
wins. Please file an issue when you find such a drift.
